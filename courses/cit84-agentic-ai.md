## 2. Updated Course Objectives (The Technical Requisite Loop)
Because this course enforces a hard prerequisite of **CIT 93 (JavaScript)**, objectives are designed to leverage programmatic logic, asynchronous events, and state arrays rather than simple conversational prompting.

| **Objective #** | **Legacy Objective** | **Proposed Objective (Agentic Alignment)** |
| :--- | :--- | :--- |
| **1** | Build a simple page in the selected framework. | Construct scalable, component-driven user interfaces using modern framework architectures. |
| **2** | Understand the framework logic and syntax. | Analyze and orchestrate framework-specific application lifecycle methods, conditional rendering, and dynamic routing states. |
| **3** | Build a CRUD web app in the selected framework. | Architect secure, relational database data integrations (utilizing **CIT 28 SQL structures**) connected to automated API endpoints. |
| **4** | *None* | **Design, provision, and configure autonomous AI agents with specialized functional toolsets (file-system execution, model chaining, and memory loops).** |
| **5** | *None* | **Implement programmatic "Human-in-the-Loop" validation gates to intercept, audit, and secure AI-generated codebase modifications before deployment.** |

## 3. Lecture & Lab Outline (The 18-Week Breakdown)

### **Unit 1: Environment & Tool Provisioning (Weeks 1-4)**
*   **Lecture Topics:** Beyond Autocomplete: The Transition from Copilot Workflows to Autonomous Agents; Introduction to Agentic Orchestration Frameworks (Vercel AI SDK / LangChain.js); Understanding Token Consumption, Latency, and Context Window Constraints.
*   **Lab Deliverables:** Configuring local execution environments (Ollama/Gemma/Llama environments); Writing basic JavaScript wrappers to initiate asynchronous model-calling loops.

### **Unit 2: State Management & Persistent Memory (Weeks 5-9)**
*   **Lecture Topics:** Managing State across Component Libraries; Short-term Session Memory vs. Long-term Database Memory; Structuring Relational Schemas (**CIT 28 SQL Integration**) to store agent conversation logs and architectural states.
*   **Lab Deliverables:** Building a framework application where an AI agent remembers previous user interactions and queries a live SQL database to pull context-specific technical answers.

### **Unit 3: Function Calling & Tool Use (Weeks 10-14)**
*   **Lecture Topics:** Teaching the Model to Act: Defining JavaScript Functions as Executable LLM Tools; Handling Model Hallucinations and Bad Syntax Outputs; Structuring Robust Error-Handling Diagnostics using AI Log Analysis.
*   **Lab Deliverables:** Developing an agentic system that is given a specific engineering goal (e.g., "Build a responsive landing page layout") and autonomously selects and executes local file-writing tools until the goal is met.

### **Unit 4: The Supervised Capstone (Weeks 15-18)**
*   **Lecture Topics:** Security Risks of Autonomous Code Execution; The Developer as the Auditor; Designing Fail-Safe Intercept Protocols.
*   **Lab Deliverables:** Final Portfolio Presentation. Students present a multi-page framework application built under the supervision of their own custom-coded AI agent loops, accompanied by a mandatory **Verbal Code Walkthrough** proving manual comprehension of all underlying files.

## 4. Methods of Evaluation & Academic Integrity
*   **Verbal Code Walkthroughs (1-on-1 Interviews):** To ensure total student authenticity in an AI-accelerated landscape, 40% of the course grade is tethered to live or recorded video explanations where students must dissect the underlying syntax, logic paths, and component trees of their submissions.
*   **Prompt Logging & Auditing Notebooks:** Students must submit their agent configuration logs alongside their code, documenting the debugging paths and structural directions they issued to the autonomous engine.
