## I. Course Description & Structural Details

### Course Title
CIT 84: Foundational Coding Concepts & Algorithmic Logic

### Catalog Description
An introductory, language-agnostic course in computer programming logic and computational thinking. Focuses on core problem-solving methodologies, algorithm development, and logical structure design using pseudocode, flowcharts, and visual execution models. Designed to prepare students for syntax-heavy scripting languages by establishing foundational programming concepts such as variables, data types, conditional structures, loops, functions, and modular design.

### Prerequisites & Corequisites
*   **Prerequisite:** CIT 48, CIT 15, and CIT 82.
*   **Advisory Corequisite:** CIT 85 (HTML/CSS), or concurrent enrollment.

## II. Core Course Content Units (18-Week Framework)

### Block 1: Computational Thinking & Problem Deconstruction (Weeks 1–4)
*   Deconstructing real-world problems into discrete, logical steps (Decomposition).
*   Pattern recognition, abstraction, and identifying essential data requirements.
*   Designing linear, step-by-step instructions without language-specific code (Algorithmic design).
*   Introduction to industry-standard flowcharts (symbols, processing blocks, decision diamonds, and terminal boundaries).

### Block 2: Data Representation & Core Structures (Weeks 5–9)
*   **Variables & Constants:** Memory slot concepts, assignment operators, initialization, and naming conventions.
*   **Data Types:** Distinguishing primitive structures (strings, integers, floats, booleans, and null/undefined values).
*   **Conditional Logic:** Single-alternative (`if`), dual-alternative (`if/else`), and multi-alternative (`switch`, nested conditions) logic flows.
*   **Boolean Operations:** Evaluating logical expressions using truth tables and boolean operators (`AND`, `OR`, `NOT`).

### Block 3: Repetition & Control Flow (Weeks 10–13)
*   **Loop Structures:** Pre-test loops (`while`), post-test loops (`do-while`), and count-controlled loops (`for`).
*   **State Management in Loops:** Managing counters, accumulators, and sentinel values to prevent infinite loop hazards.
*   **Dry-Running Logic:** Manually tracing and documenting changing variable states over time using trace tables.

### Block 4: Modular Design & The "Human-in-the-Loop" (Weeks 14–18)
*   **Functions & Modules:** Defining functional inputs (parameters) and structural outputs (return values).
*   **Variable Scope:** Differentiating between local and global data boundaries.
*   **AI Orchestration Literacy:** Utilizing LLMs to generate alternative algorithmic approaches or debug pseudocode logic, with a strict focus on critical evaluation and manual verification of AI-generated workflows.

## III. Core Pedagogy: The "Mechanical Shop" Alignment

*   **No Copy-Paste Prompting:** If AI tools are integrated into a discovery exercise, students must map the resulting logic back to a visual flowchart or custom pseudocode structure to prove conceptual mastery of the execution path.
*   **Trace Tables Over Compilers:** Rather than relying on a software compiler or console to catch errors, core logic assessments require manually stepping through code paths with dry-run tables to calculate exact variable outputs.
*   **Low-Stakes Verbal Code Reviews:** Introducing early-stage verbal assessments where students verbally explain logic flows (e.g., *“If this variable meets this condition, the logic branches down this path because...”*) to build professional confidence and defensive code-literacy skills.
