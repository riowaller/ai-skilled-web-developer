# Student Readiness & "Bridge" System Plan

## 🎯 Concept: Supportive Entry
To balance **Enrollment Accessibility** (specifically for CSU Transfer students) with **Technical Rigor**, the team will implement a "Supportive Entry" model. This replaces hard prerequisites with a proactive "Skill-Up" system to ensure all students are prepared for the "Mechanical Shop" environment.

---

## 🤝 The "Post-82" Technical Handshake
Courses that follow CIT 82 (CIT 93, CIT 28, CIT 94, and CIT 84) operate under the assumption that students have mastered foundational workflow tools. The **Success Audit** identifies gaps for students who may have bypassed CIT 82.

### **1. Git & Version Control**
* **The Requirement:** Mastery of the basic local-to-remote workflow (Clone, Commit, Push).
* **The Audit:** "Can you clone a repository and push your code to GitHub via the terminal or VS Code?"
* **The Bridge:** Students must complete a "Git for Developers" mini-bootcamp in Module 0.
* **CIT 28 Specifics:** Ensuring students can version-control their database schemas and `.sql` migration scripts.

### **2. Terminal & Environment Literacy**
* **The Requirement:** Basic Command Line Interface (CLI) proficiency.
* **The Audit:** "Are you comfortable navigating directories and running scripts using the terminal?"
* **The Bridge:** A "CLI Quick-Start" guide and environment setup checklist for VS Code.

### **3. AI Troubleshooting Protocol**
* **The Requirement:** Using AI as a peer-tutor rather than a code generator.
* **The Audit:** "Do you know how to prompt an LLM to explain a logic error or SQL syntax issue without asking it to write the solution?"
* **The Bridge:** A "Prompting for Success" guide that sets the program's academic integrity expectations.

## 🔄 The Universal Bridge Workflow

| Phase | Component | Action |
| :--- | :--- | :--- |
| **Step 1** | **The Success Audit** | Standardized Canvas survey available 2 weeks prior to start. |
| **Step 2** | **The Skill-Up Roadmap** | Automated email based on scores (Green, Yellow, Red). |
| **Step 3** | **Module 0 (The Bridge)** | Targeted "Gap-Filler" content (Git, CLI, AI Ethics). |
| **Step 4** | **The Day 1 Huddle** | Focused "Week 0" office hour for environment setup. |

---

## 🧠 Strategic Rationale
1.  **Protects Enrollment:** Maintains accessibility for CSU transfer students by avoiding hard prerequisites that add extra units to their path.
2.  **Standardizes Submission:** If every course—including **CIT 28 (SQL)**—uses GitHub for labs, the grading workflow becomes faster and students gain professional-grade habits.
3.  **Reduces Friction:** Minimizes the first two weeks of "tech support" overhead for faculty, allowing the class to dive immediately into core concepts.
