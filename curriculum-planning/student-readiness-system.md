# Student Readiness & "Bridge" System Plan

## 🎯 Concept: Supportive Entry & Proactive Inclusion
To balance **Enrollment Accessibility** (specifically for CSU Transfer students) with **Technical Rigor**, the program utilizes a "Supportive Entry" model. This replaces restrictive gatekeeping with **Strongly Recommended Advisories** and a proactive "Skill-Up" system. 

By aligning with **Title 5 § 55001**, this system ensures that students with varied preparation levels receive the specific "Hidden Curriculum" knowledge—such as terminal literacy and Git workflows—needed to succeed in the "Mechanical Shop" environment.

---

## 🤝 The "Technical Handshake" (Success Audit)
Courses in the pathway assume students possess specific foundational workflow tools. If a student chooses to bypass the recommended advisory sequence, they must demonstrate competency in these **3 Minimum Day-1 Skills** during Module 0.

### **CIT 82: Intro to AI & Web**
*Bridge for those skipping CIT 48A or CIT 15*
1. **OS File Management:** Can navigate a local directory structure and manage file extensions (e.g., .html, .css).
2. **Web Literacy:** Understands the distinction between a local file path and a live URL.
3. **Hardware Readiness:** Verified access to a machine capable of running local dev environments (VS Code).

### **CIT 85: Applied HTML/CSS**
*Bridge for those skipping CIT 82*
1. **VS Code Environment:** Can open a project folder and utilize "Live Server" for local testing.
2. **AI Ethic Mastery:** Explicitly understands the "Human-in-the-Loop" policy (AI for ideation, manual authoring for labs).
3. **Syntax Awareness:** Can identify the fundamental structure of an HTML element without AI assistance.

### **CIT 28: SQL & CIT 93: JavaScript**
*Bridge for those skipping CIT 85*
1. **Terminal Literacy:** Can navigate project directories via the Command Line Interface (CLI).
2. **Git/GitHub Mastery:** Can perform a basic `clone`, `commit`, and `push` to a repository.
3. **Logical Troubleshooting:** Can describe a code error to an AI tutor to receive a *guided explanation* rather than a code dump.

### **CIT 94: AI Web Workflows (Capstone)**
*Bridge for those skipping CIT 93*
1. **Logic/State Foundations:** Understands variables, functions, and the concept of "state" in an application.
2. **API/Data Literacy:** Understands the "Handshake" between a client request and a server response.
3. **Security Awareness:** Understands how to secure sensitive API keys using `.env` files.

## 🔄 The Universal Bridge Workflow

| Phase | Component | Action |
| :--- | :--- | :--- |
| **Step 1** | **The Success Audit** | Canvas diagnostic quiz available 2 weeks prior to start. |
| **Step 2** | **The Skill-Up Roadmap** | Automated feedback directing students to specific "Bridge" labs based on audit gaps. |
| **Step 3** | **Module 0 (The Bridge)** | Targeted technical content (Git, CLI, AI Ethics) completed by Day 1. |
| **Step 4** | **The Verbal Check** | Instructors use the first lab session to verify the "Technical Handshake" in person. |

---

## 🧠 Strategic Rationale
1. **Protects Enrollment:** Maintains accessibility for CSU transfer students by avoiding hard prerequisites that add unnecessary units.
2. **Equity in Action:** Directly supports students from under-resourced technical backgrounds by providing "on-ramp" instruction for the modern dev environment.
3. **Instructor Efficiency:** Reduces "Technical Debt" in later courses by ensuring all students speak the same "Terminal Language" before the first lecture.
