# Student Readiness & "Bridge" System Plan

## 🎯 Concept: Supportive Entry
To balance **Enrollment Accessibility** (specifically for CSU Transfer students) with **Technical Rigor**, the program utilizes a "Supportive Entry" model. This replaces hard prerequisites with **Strongly Recommended Advisories** and a proactive "Skill-Up" system to ensure all students are prepared for the "Mechanical Shop" environment.

---

## 🤝 The "Technical Handshake" (Success Audit)
Courses in the pathway operate under the assumption that students possess specific foundational workflow tools. If a student chooses to bypass the recommended advisory sequence, they must demonstrate competency in these **3 Minimum Day-1 Skills** during Module 0.

### **CIT 82: Intro to AI & Web**
*Bridge for those skipping CIT 48A or CIT 15*
1. **File Management:** Can create, rename, and move folders/files within a local OS.
2. **Web Navigation:** Can use browser DevTools (Inspect) to identify page elements.
3. **Digital Literacy:** Understands the difference between local storage and cloud syncing.

### **CIT 85: Applied HTML/CSS**
*Bridge for those skipping CIT 82*
1. **VS Code Basics:** Can open a project folder and navigate the integrated terminal.
2. **AI Ethics:** Understands the "Mechanical Shop" policy (AI as a tutor, not a writer).
3. **Document Structure:** Recognizes the purpose of a standard HTML boilerplate.

### **CIT 28: SQL & CIT 93: JavaScript**
*Bridge for those skipping CIT 82 & 85*
1. **Git/GitHub Mastery:** Can perform a basic `clone`, `commit`, and `push` to a repository.
2. **Terminal Literacy:** Can navigate project directories via the Command Line Interface (CLI).
3. **Debugging Protocol:** Can use `console.log` or SQL error logs alongside AI to identify syntax errors.

### **CIT 94: AI Web Workflows (Capstone)**
*Bridge for those skipping CIT 93*
1. **Logic Foundations:** Understands variables, functions, and asynchronous flows.
2. **API Literacy:** Understands the fundamental concept of a "Request" and "Response."
3. **Environment Security:** Understands how to secure sensitive API keys using `.env` files.

## 🔄 The Universal Bridge Workflow

| Phase | Component | Action |
| :--- | :--- | :--- |
| **Step 1** | **The Success Audit** | Canvas diagnostic quiz available 2 weeks prior to start. |
| **Step 2** | **The Skill-Up Roadmap** | Automated feedback directing students to specific "Bridge" labs based on audit gaps. |
| **Step 3** | **Module 0 (The Bridge)** | Targeted technical content (Git, CLI, AI Ethics) completed by Day 1. |
| **Step 4** | **The Day 1 Huddle** | A focused "Week 0" office hour session to finalize environment setups. |
