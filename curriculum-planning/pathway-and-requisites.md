## 🛣️ The "Student Success" Map (Recommended 3-Semester Sequence)
*Note: **CIT 15** is the foundational prerequisite for the program and is assumed to be completed prior to Semester 1.*

### **Semester 1: The Foundations**
* **CIT 82:** Introduction to AI and Web Development.
* **CIT 85:** Applied HTML and CSS.
* *Goal: Establish AI literacy and manual authoring "Mechanical Shop" basics.*

### **Semester 2: Programming Fundamentals**
* **CIT 93:** JavaScript.
* *Goal: Focus exclusively on logic, TDD, and asynchronous programming.*

### **Semester 3: Advanced Data & Integration**
* **CIT 28:** SQL (Data Modeling).
* **CIT 94:** AI Web Workflows (Capstone).
* *Goal: Combine logic with data management and AI API orchestration.*

## 🧠 Workgroup Rationale & Challenges

### 1. The "JavaScript Barrier"
Faculty have observed students struggling in CIT 93 without prior coding exposure. By requiring CIT 82 first, we ensure students have developed troubleshooting skills and web context before tackling complex JS logic.

### 2. The CSU Fresno Transfer Dilemma
* **The Conflict:** CIT 93 is a popular service course for students outside our program because it transfers to CSU Fresno for graduation requirements. 
* **The Concern:** Adding CIT 82 as a hard prerequisite creates a "two-course hurdle" for transfer students who may only need JavaScript. This could negatively impact CIT 93 enrollment.
* **Discussion Points:** * Should we use a "Strongly Recommended Advisory" for CIT 82 instead of a Prerequisite?
    * Can we implement a prerequisite challenge process for transfer students with equivalent experience?

### 3. Separation of CIT 93 and CIT 28
To improve student retention, the plan separates these two high-rigor courses. Teaching JavaScript and SQL in the same term often leads to cognitive overload; this sequence allows for a focused deep-dive into each domain.
